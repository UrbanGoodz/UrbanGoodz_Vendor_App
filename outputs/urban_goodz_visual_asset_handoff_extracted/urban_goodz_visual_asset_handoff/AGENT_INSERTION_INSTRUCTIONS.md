# Urban Goodz Visual Asset Handoff

## Purpose
This ZIP contains the current Urban Goodz visual asset direction and reference artwork for feature cards, category cards, buttons, and pathway-opening modules.

This is a DESIGN HANDOFF ONLY.

Do not create code from these files.
Do not redesign the app UI.
Do not replace existing category artwork unless explicitly instructed.
Do not invent a new Urban Goodz brand style.

The agent should use these assets as the visual direction/source references for building or inserting standalone square artwork/assets into the Urban Goodz app and marketing library.

---

## Absolute Brand Rules

Use the Urban Goodz palette only:

- Seasoning Orange: `#ED9914`
- Canvas: `#E2D3BF`
- Dijon: `#E5E276`
- UG Black: `#161616`
- White: `#FFFFFF`

Typography direction:

- Main labels should feel rounded, modern, premium, and urban-friendly, similar to Thyga Semirounded.
- Secondary text should feel clean and readable, similar to Arial Nova.
- Keep text minimal and large enough to read.
- Do not create tiny unreadable UI copy.
- Do not distort text.

Logo rules:

- Urban Goodz name/logo must NOT sit directly on a black background.
- Use the UG mark as a small, controlled brand element only.
- Acceptable placements: corner badge, key tag, package mark, provider card, request card, loyalty card, rental tag, membership card, route card, search bubble, or subtle watermark.
- Avoid giant centered logos, glowing UG logos, repeated logo spam, or fake alternate logos.

Background rules:

- Prefer Canvas, White, Dijon, and light warm backgrounds.
- Dark backgrounds should only be used for small objects/cards, not behind the Urban Goodz wordmark/name.
- Do not return to the earlier dark neon logo-heavy style.

---

## Core Visual Requirement

Every visual asset must clearly explain the feature before the user reads the label.

Each asset should answer:

1. What is this feature?
2. What action can the customer or provider take?
3. What transaction, booking, delivery, earning, rental, service, or local action does it represent?

Use clear visual routes:

- Search → Results → Action
- Request → Match → Completion
- Need → Provider → Service Done
- Product → Creator Tag → Sale
- Driver Opportunity → Route → Payout
- Medical Pickup → Secure Transport → Lab/Clinic Drop-off
- Search Nearby → Choose Car → Book/Reserve → Pick Up & Go

Use step cards, arrows, route lines, numbered process chips, connected nodes, or labeled object sequences.

Avoid floating random objects with no purpose.

---

## Critical Rentals Correction

Urban Goodz Rentals should focus on CAR RENTALS, not home rentals.

Use:

- Cars
- SUVs
- Trucks
- Keys
- Rental access card
- Booking calendar
- pickup/return time
- insurance options
- daily/weekly rates
- roadside support
- pickup route or map cue

Do NOT use:

- Airbnb-style home rentals
- short-term apartment stays
- hotel/room cards
- home interiors
- event space as the main rental story

The correct route for Rentals Near You is:

Search Nearby → Choose & Compare Vehicles → Book & Reserve → Pick Up & Go

The viewer should instantly understand:

Urban Goodz helps me find nearby cars, trucks, SUVs, and rental vehicles.

## Final Asset Standards

Each final image should be:

- 1:1 square format
- standalone
- brand-consistent
- visually specific
- functionally clear
- not dark/neon heavy
- not logo-heavy
- not generic
- not repetitive
- light or mixed-background unless there is a strong reason otherwise
- clear enough that users understand the feature before reading the title

The final output should feel like:

Urban Goodz brand system + clear feature education + premium app artwork.
